## Algorithm

**Isolation Forest** is an unsupervised anomaly detection algorithm based on the principle that anomalies are *few* and *different*:

- Builds an ensemble of random **Isolation Trees**
- Each tree randomly partitions the feature space
- **Anomalies are isolated in fewer splits** → shorter average path length
- Anomaly score = normalised average path length across all trees
- No distance or density estimation needed — very efficient

### Key Hyperparameters Used

| Parameter | Value | Reason |
|---|---|---|
| n_estimators | 200 | More trees → stable scores |
| contamination | ~0.0017 | Matches true fraud proportion |
| max_samples | auto (256) | Standard practice |
| max_features | 1.0 | Use all features per split |
| bootstrap | False | Without-replacement sampling |

## Pipeline

```
Raw CSV
  └─► Drop 'Time' column
        └─► StandardScaler (29 features)
              └─► Isolation Forest (200 trees)
                    └─► Predictions + Anomaly Scores
                          └─► Evaluate vs ground-truth labels
```

## Results

| Metric | Value |
|---|---|
| Precision | ~0.28 |
| Recall | ~0.27 |
| F1 Score | ~0.27 |
| ROC-AUC | ~0.94 |

> Note: Precision/Recall/F1 are low by design — Isolation Forest is fully **unsupervised** and receives no fraud labels during training. The high ROC-AUC (~0.94) shows the anomaly scores strongly rank fraud above normal transactions. For a labelled task, supervised models will outperform.

- **Feature Importance**: Permutation-based importance on fraud samples
